បញ្ញាសិប្បនិម្មិត (Artificial Intelligence ឬ AI) គឺជាបច្ចេកវិទ្យាមួយក្នុងវិទ្យាសាស្ត្រកុំព្យូទ័រ ដែលផ្តោតលើការបង្កើតប្រព័ន្ធ ឬកម្មវិធីដែលអាចធ្វើការសម្រេចចិត្ត រៀនពីទិន្នន័យ វិភាគព័ត៌មាន និងដោះស្រាយបញ្ហាបានដូចមនុស្ស។ AI អាចស្គាល់រូបភាព ស្តាប់សំឡេង បកប្រែភាសា ឆ្លើយសំណួរ និងទស្សន៍ទាយលទ្ធផលដោយផ្អែកលើទិន្នន័យដែលបានរៀន។ បច្ចុប្បន្ន AI ត្រូវបានប្រើប្រាស់យ៉ាងទូលំទូលាយក្នុងវិស័យសុខាភិបាល ការអប់រំ ធនាគារ កសិកម្ម ពាណិជ្ជកម្ម និងឧស្សាហកម្មជាច្រើន។

២. ការពិចារណា និងសម្រេចចិត្ត (Reasoning & Decision Making)

- Logical Reasoning, Probabilistic Reasoning និង Decision Trees គឺជាវិធីសាស្រ្តសំខាន់ៗសម្រាប់ការពិចារណា និងសម្រេចចិត្ត។

AI អាចវិភាគទិន្នន័យដើម្បីរកលំនាំ និងទស្សន៍ទាយលទ្ធផល។ វាត្រូវបានប្រើក្នុងការវិភាគអតិថិជន ទស្សន៍ទាយទីផ្សារ និងស្វែងរកការប្រព្រឹត្តបទល្មើស។ ឧទាហរណ៍៖ ប្រព័ន្ធ AI អាចវិភាគទិន្នន័យអតិថិជនដើម្បីផ្តល់អនុសាសន៍ផលិតផលឬសេវាកម្មដែលពួកគេចូលចិត្ត។ - Natural Language Processing (NLP) និង Computer Vision គឺជាវិធីសាស្រ្តសំខាន់ៗសម្រាប់ការវិភាគទិន្នន័យ។

AI អាចធ្វើសកម្មភាពដោយស្វ័យប្រវត្តិ ដោយគ្មានការគ្រប់គ្រងពីមនុស្សដោយផ្អែកលើលទ្ធផលដែលបានពី Learning,Reasesioning,Analysis។ ឧទាហរណ៍៖ Robot នៅរោងចក្រ, Self-driving car, និងប្រព័ន្ធណែនាំក្នុងអ៊ីនធឺណិតដែលធ្វើការដោយស្វ័យប្រវត្តិ។

សរុប

មុខងារសំខាន់ៗរបស់ AI រួមមាន ការរៀនពីទិន្នន័យ(Learning) + ការពិចារណា(Reasesioning) + ការវិភាគ(Analysing) + និងការអនុវត្តសកម្មភាព។ មុខងារទាំងនេះធ្វើឲ្យ AI ក្លាយជាបច្ចេកវិទ្យាដ៏សំខាន់ក្នុងសម័យឌីជីថល និងជួយបង្កើនប្រសិទ្ធភាពក្នុងជីវិតប្រចាំថ្ងៃ។

មេរៀនទី៣

របៀបដំណើរការរបស់បញ្ញាសិប្បនិម្មិត (AI Workflow)

AI ដំណើរការតាមជំហានជាបន្តបន្ទាប់ពីទិន្នន័យទៅការរៀន បណ្តុះបណ្តាល សម្រេចចិត្ត និងការកែលម្អ។ ជំហានទាំងនេះជួយឲ្យ AI កាន់តែឆ្លាត និងមានភាពត្រឹមត្រូវខ្ពស់។

1. 📊 ទិន្នន័យ (Data Collection)

ដំណើរការរបស់ AI ចាប់ផ្តើមពីទិន្នន័យ។ ទិន្នន័យអាចជារូបភាព វីដេអូ សំឡេង អត្ថបទ ឬព័ត៌មានពីអ្នកប្រើប្រាស់។ ទិន្នន័យទាំងនេះត្រូវបានប្រមូលពីប្រភពផ្សេងៗ ហើយយកមកសម្អាត (cleaning) និងរៀបចំ (preprocessing) ដើម្បីឲ្យម៉ាស៊ីនអាចយល់បាន។ ប្រសិនបើទិន្នន័យមានកំហុស ឬលំអៀង នោះ AI នឹងរៀនខុសដែរ។

⬇

2. 🧠 ការរៀន (Learning)

នៅដំណាក់កាលនេះ AI ចាប់ផ្តើមស្វែងរកលំនាំ (patterns) និងទំនាក់ទំនងក្នុងទិន្នន័យ។ វាប្រើ algorithm ដើម្បីយល់ថា តើអ្វីជាអ្វី។ ឧទាហរណ៍ AI អាចរៀនថា រូបភាពដែលមានភ្នែក ច្រមុះ និងមាត់ អាចជាមុខមនុស្ស។ នេះគឺជាដំណាក់កាលដែល AI បង្កើត “ចំណេះដឹងដំបូង” របស់ខ្លួន។

⬇

3. ⚙️ ការបណ្តុះបណ្តាលម៉ូដែល (Model Training)

ម៉ូដែល (Model) គឺជាគណិតវិទ្យា ឬ algorithm ដែល AI ប្រើសម្រាប់ទស្សន៍ទាយ។ ក្នុងដំណាក់កាលនេះ AI ត្រូវបានបណ្តុះបណ្តាលដោយទិន្នន័យច្រើនដង ដើម្បីកាត់បន្ថយកំហុស (error) និងបង្កើនភាពត្រឹមត្រូវ។ វាធ្វើការកែសម្រួលខ្លួនឯងជាបន្តបន្ទាប់រហូតដល់ទទួលបានលទ្ធផលល្អ។

⬇

4. 🎯 ការសម្រេចចិត្ត និងសកម្មភាព (Decision Making & Action)

បន្ទាប់ពីម៉ូដែលត្រូវបានបណ្តុះបណ្តាល AI អាចប្រើវាដើម្បីសម្រេចចិត្ត។ ឧទាហរណ៍ AI អាចស្គាល់រូបភាពថាជាឆ្មា ឬឆ្កែ, ឬសម្រេចថាប្រតិបត្តិការធនាគារមានសុវត្ថិភាពឬអត់។ នៅក្នុងប្រព័ន្ធពិត AI អាចធ្វើសកម្មភាពដោយស្វ័យប្រវត្តិ ដូចជា បើកបររថយន្ត ឆ្លើយសំណួរ ឬណែនាំផលិតផល។

⬇

5. 🔄 ការកែលម្អជាបន្តបន្ទាប់ (Continuous Improvement)

AI មិនឈប់រៀនទេ។ នៅពេលវាត្រូវបានប្រើប្រាស់ក្នុងពិភពពិត វានឹងទទួលបានទិន្នន័យថ្មីៗ និង feedback ពីអ្នកប្រើប្រាស់។ ទិន្នន័យថ្មីៗនេះត្រូវបានយកទៅបន្ថែមក្នុងការបណ្តុះបណ្តាលវិញ ដើម្បីធ្វើឲ្យម៉ូដែលកាន់តែឆ្លាត និងត្រឹមត្រូវជាងមុន។ ដំណើរការនេះហៅថា “Feedback Loop” ដែលធ្វើឲ្យ AI កែលម្អខ្លួនឯងជាបន្តបន្ទាប់។

មេរៀនទី៤

ប្រភេទនៃបញ្ញាសិប្បនិម្មិត (Types of AI)

បញ្ញាសិប្បនិម្មិត (AI) អាចត្រូវបានបែងចែកជាប្រភេទផ្សេងៗ ដោយផ្អែកលើកម្រិតនៃភាពឆ្លាត និងសមត្ថភាពក្នុងការគិត។ ប្រភេទទាំងនេះបង្ហាញពីការវិវត្តន៍ពី AI សាមញ្ញទៅ AI ដែលមានសមត្ថភាពខ្ពស់។ 

Narrow AI (AI តូច / AI មានកម្រិត)

Narrow AI គឺជា AI ដែលត្រូវបានបង្កើតឡើងសម្រាប់ការងារតែមួយប៉ុណ្ណោះ។ វាមិនអាចគិតក្រៅពីភារកិច្ចដែលវាត្រូវបានបណ្តុះបណ្តាល។ ឧទាហរណ៍៖ Face Recognition, Google Translate, Chatbot និង YOLO Object Detection។ AI ប្រភេទនេះគឺជាប្រភេទដែលយើងប្រើប្រាស់ជាទូទៅបំផុតក្នុងពិភពបច្ចុប្បន្ន។


General AI (AI ទូទៅ)

General AI គឺជា AI ដែលមានសមត្ថភាពគិត និងរៀនដូចមនុស្ស។ វាអាចធ្វើការងារជាច្រើនប្រភេទ មិនចាំបាច់កំណត់តែមួយភារកិច្ចទេ។ ឧទាហរណ៍ វាអាចរៀនគណិតវិទ្យា សរសេរកូដ និងយល់ពីអារម្មណ៍មនុស្សបាន។ ប៉ុន្តែបច្ចុប្បន្ន General AI នៅតែជាគំនិត និងការស្រាវជ្រាវប៉ុណ្ណោះ។


​Super AI (AI លើសមនុស្ស)

Super AI គឺជា AI ដែលមានសមត្ថភាពខ្ពស់ជាងមនុស្សក្នុងគ្រប់វិស័យ ដូចជា ការគិត វិភាគ សម្រេចចិត្ត និងការច្នៃប្រឌិត។ វាអាចដោះស្រាយបញ្ហាស្មុគស្មាញបានលឿនជាងមនុស្សជាច្រើនដង។ បច្ចុប្បន្ន Super AI មិនទាន់មានជាក់ស្តែងទេ ប៉ុន្តែជាគោលដៅនាពេលអនាគត។

💡 សរុប៖ AI មានការវិវត្តន៍ពី Narrow AI → General AI → Super AI

YOLO (You Only Look Once)

YOLO គឺជា Model សម្រាប់រកវត្ថុ (Object Detection) ដែលអាចមើលរូបភាពតែមួយដងប៉ុណ្ណោះ ហើយអាចទាយបានទាំងទីតាំង (Location) និងប្រភេទវត្ថុ (Class) ក្នុងពេលតែមួយ។ វាធ្វើឲ្យប្រព័ន្ធមានល្បឿនលឿនខ្លាំង និងអាចប្រើបានក្នុងការងារពេលវេលាពិត (Real-time) ដូចជា Camera, Self-driving car និង AI Surveillance។

ហេតុអ្វី YOLO លឿន?

មុននេះ Object Detection ប្រើវិធីច្រើនជំហាន (Multi-stage) ដូចជា៖ ស្វែងរក Region ជាមុន → បន្ទាប់មកចាត់ថ្នាក់វត្ថុក្នុង Region នោះ។ ប៉ុន្តែ YOLO បម្លែងបញ្ហានេះទៅជា បញ្ហា Regression តែមួយដង ដោយទាយ bounding boxes និង class ទាំងអស់ក្នុងការបញ្ជូនតែមួយ (single forward pass)។ ដូច្នេះវាលឿនជាងប្រព័ន្ធចាស់ៗយ៉ាងខ្លាំង។

One-Stage Detector (ប្រព័ន្ធមួយជំហាន)

YOLO បែងចែករូបភាពទៅជា Grid (តារាងតូចៗ)។ Grid នីមួយៗមានភារកិច្ចទាយថា៖ តើមានវត្ថុឬអត់? បើមាន វានឹងទាយ bounding box និង class probability ភ្លាមៗ។ មិនចាំបាច់មានការបង្កើត Region Proposal ដូច R-CNN ទេ។

Bounding Box និង Confidence Score

Bounding Box គឺជាស៊ុម (Rectangle) ដែលបង្ហាញទីតាំងវត្ថុក្នុងរូបភាព។ វាមានតម្លៃដូចជា៖ x, y (ទីតាំង) និង width, height (ទំហំ)។ YOLO ក៏ផ្តល់ Confidence Score ដែលបង្ហាញកម្រិតជឿជាក់ថា នៅក្នុង box នោះមានវត្ថុពិតប្រាកដ។

Class Prediction និង NMS

ក្នុងមួយ Bounding Box YOLO ក៏ទាយប្រភេទវត្ថុ (Class Probability) ដូចជា មនុស្ស, ឡាន, ឆ្កែ។ បន្ទាប់មកប្រើ Non-Maximum Suppression (NMS) ដើម្បីលុប box ដែលស្ទួនគ្នា ហើយរក្សាទុកតែ box ដែលមាន confidence ខ្ពស់បំផុតសម្រាប់វត្ថុនីមួយៗ។

ស្ថាបត្យកម្ម YOLO (YOLO Architecture)

ម៉ូឌែល YOLO ត្រូវបានបែងចែកជាបីផ្នែកសំខាន់ ដែលធ្វើការជាមួយគ្នា ដើម្បីអាចសម្គាល់វត្ថុបានលឿន និងត្រឹមត្រូវ។

01

Backbone (ផ្នែកទាញយកលក្ខណៈ)

Backbone គឺជាបណ្តាញ Convolutional Neural Network (CNN) ដូចជា CSPDarknet។ វាមានភារកិច្ចក្នុងការទាញយកលក្ខណៈសំខាន់ៗពីរូបភាព ដូចជា៖

  • គែម (Edges)
  • រចនាសម្ព័ន្ធ និងប្លង់ (Shapes & Structures)
  • វាយនភាព (Textures)
  • លក្ខណៈពិសេសសំខាន់ៗរបស់វត្ថុ
02

Neck (ផ្នែកបញ្ចូលលក្ខណៈច្រើនកម្រិត)

Neck ជាស្រទាប់មធ្យមដែលភ្ជាប់ Backbone និង Head។ វាប្រើបច្ចេកទេសដូចជា FPN (Feature Pyramid Network) និង PANet ដើម្បីបញ្ចូលព័ត៌មានពីកម្រិតផ្សេងៗនៃរូបភាព។

  • បញ្ចូលព័ត៌មានពីរូបភាពធំ និងតូច
  • ជួយរកវត្ថុតូចៗបានល្អជាងមុន
  • បង្កើនភាពត្រឹមត្រូវក្នុងការសម្គាល់វត្ថុ
03

Head (ផ្នែកទស្សន៍ទាយលទ្ធផល)

Head គឺជាផ្នែកចុងក្រោយដែលធ្វើការទស្សន៍ទាយលទ្ធផល។ វាបម្លែងលក្ខណៈដែលបានទាញយកទៅជាព័ត៌មានសម្រាប់ការសម្គាល់វត្ថុ។

  • Bounding Box (ប្រអប់កំណត់ទីតាំងវត្ថុ)
  • Confidence Score (កម្រិតទំនុកចិត្ត)
  • Class Probability (ប្រភេទវត្ថុ)

LOCATING OBJECTS

The Bounding Box

Bounding Box គឺជាគំនិតសំខាន់មួយក្នុង Object Detection ដែលប្រើសម្រាប់កំណត់ទីតាំងរបស់វត្ថុនៅក្នុងរូបភាព។ YOLO មិនត្រឹមតែអាចដឹងថាវត្ថុជា​អ្វីទេ ប៉ុន្តែវាក៏អាចកំណត់ថាវត្ថុនោះស្ថិតនៅត្រង់ណា និងមានទំហំប៉ុនណា។

នៅក្នុងប្រព័ន្ធ AI មួយ Bounding Box ត្រូវបានប្រើដើម្បី “គូសប្រអប់” ជុំវិញវត្ថុដែលបានរកឃើញ។ ប្រអប់នេះអាចផ្លាស់ប្តូរទំហំ និងទីតាំងបាន ដើម្បីឲ្យសមនឹងវត្ថុពិតក្នុងរូបភាព។

រាល់ Bounding Box មួយត្រូវបានបង្កើតឡើងដោយលេខសំខាន់ ៤ គេហៅថា x, y, w, h ដែលជាគន្លងគណិតវិទ្យាសម្រាប់កំណត់ទីតាំង និងទំហំ។

x — ទីតាំងកណ្ដាលតាមអ័ក្ស X (horizontal)
បង្ហាញថាវត្ថុស្ថិតនៅខាងឆ្វេង ឬស្តាំក្នុងរូបភាពប៉ុណ្ណា
y — ទីតាំងកណ្ដាលតាមអ័ក្ស Y (vertical)
បង្ហាញថាវត្ថុស្ថិតនៅខាងលើ ឬខាងក្រោមក្នុងរូបភាព
w — ទទឹងប្រអប់ (width)
កំណត់ថាវត្ថុមានទំហំទូលាយប៉ុនណា តាមទិសផ្ដេក
h — កម្ពស់ប្រអប់ (height)
កំណត់ថាវត្ថុមានកម្ពស់ប៉ុនណា តាមទិសបញ្ឈរ

ក្នុង YOLO មួយរូបភាពអាចមាន Bounding Box ច្រើន។ ម៉ូដែលនឹងបង្កើតប្រអប់ជាច្រើនសម្រាប់វត្ថុដែលអាចមាន។ បន្ទាប់មកវានឹងគណនាពិន្ទុទំនុកចិត្ត (confidence score) ហើយជ្រើសរើសប្រអប់ដែលត្រឹមត្រូវបំផុត។ ប្រអប់ស្ទួនៗនឹងត្រូវបានលុបចេញដោយប្រើវិធី Non-Max Suppression (NMS)។

សរុប៖ Bounding Box គឺជារបៀបដែល AI “គូសប្រអប់” ជុំវិញវត្ថុ ដើម្បីកំណត់ទីតាំង និងទំហំរបស់វាឲ្យច្បាស់លាស់។

HOW SURE IS THE MODEL?

Confidence Score

Confidence Score គឺជាតម្លៃលេខ (0 ដល់ 1) ដែល AI ប្រើដើម្បីវាស់កម្រិត “ភាពជឿជាក់” ក្នុងការទស្សន៍ទាយវត្ថុមួយនៅក្នុងរូបភាព។ តម្លៃកាន់តែខ្ពស់ មានន័យថា ម៉ូដែលកាន់តែប្រាកដថាការទស្សន៍ទាយរបស់វាត្រឹមត្រូវ។

ក្នុង Object Detection ដូចជា YOLO រាល់ Bounding Box មួយមិនមែនមានតែទីតាំងទេ ប៉ុន្តែវាក៏ភ្ជាប់ជាមួយ “ពិន្ទុភាពជឿជាក់” ផងដែរ។ ពិន្ទុនេះគឺជាការរួមបញ្ចូលរវាងការស្គាល់ថា “មានវត្ថុឬអត់” និង “វត្ថុនោះជាអ្វី”។

ដំណើរការនេះកើតឡើងដោយម៉ូដែល Neural Network ដែលរៀនពី data រាប់លានរូបភាព។ ពេលវាមើលរូបភាពថ្មី វាប្រៀបធៀបលក្ខណៈ (features) ដែលវាធ្លាប់រៀន ហើយប៉ាន់ស្មានថា តើវត្ថុនោះស្រដៀងនឹងអ្វីដែលវាធ្លាប់ឃើញ។

  • 1. Objectness Score (មានវត្ថុឬអត់)
    វាជាតម្លៃដែលបង្ហាញថា ក្នុង Bounding Box មួយ មានវត្ថុពិតប្រាកដឬអត់។ បើ score ខ្ពស់ → មានវត្ថុច្បាស់លាស់ បើ score ទាប → អាចជាផ្ទៃទទេ ឬ noise
  • 2. Class Probability (ប្រភេទវត្ថុ)
    បន្ទាប់ពីដឹងថាមានវត្ថុហើយ ម៉ូដែលនឹងទស្សន៍ទាយថាវាជាអ្វី ដូចជា person, car, dog, bicycle... វាបង្ហាញ “សក្តានុពល” នៃប្រភេទនីមួយៗ។
  • 3. Final Confidence Score
    គឺការបូកបញ្ចូល Objectness × Class Probability ដើម្បីទទួលបានពិន្ទុចុងក្រោយសម្រាប់ Bounding Box មួយ។ នេះជាតម្លៃដែល YOLO ប្រើសម្រាប់សម្រេចថាត្រូវបង្ហាញ ឬលុបចោល។
  • 4. Confidence Threshold (កម្រិតកាត់ចោល)
    ជាកម្រិតកំណត់សម្រាប់បង្ហាញលទ្ធផល។ ប្រអប់ដែលមាន confidence ទាបជាង threshold នឹងត្រូវបដិសេធ។ វាជួយកាត់បន្ថយ false detection និង noise។

4️⃣ អ្វីកើតឡើងក្នុង ១ Epoch

  1. Shuffle Data

    លាយលំដាប់រូបភាព ដើម្បីកុំឲ្យ model ចងចាំលំដាប់

  2. Forward Pass

    បញ្ចូលរូប → model ទស្សន៍ទាយ bounding boxes + classes

  3. Compute Loss
    • Box Loss (CIoU) → កំហុសទីតាំង និងទំហំប្រអប់
    • Class Loss (BCE) → កំហុសប្រភេទ Class
    • DFL Loss → កំហុសក្នុងការកែតម្រូវប្រអប់
  4. Backward Pass (Backpropagation)

    គណនា gradient ដោយប្រើ Chain Rule ពី Calculus

  5. Optimizer Step

    កែតម្រូវ weights ដោយប្រើ SGD/Adam

  6. Validation

    ពិនិត្យលទ្ធផលលើ validation set (រៀងរាល់ epoch)

📐

5️⃣ Loss Functions លម្អិត

Box Loss (CIoU)
CIoU = 1 - IoU + (ρ²(b,b_gt)/c²) + αv
IoU = Intersection over Union
ρ = distance between centers
αv = aspect ratio penalty
Class Loss (BCE)
BCE = -[y·log(ŷ) + (1-y)·log(1-ŷ)]
វាស់កំហុសរវាង Class ពិត និងទស្សន៍ទាយ
DFL Loss
DFL = -Σ(y_i · log(ŷ_i))
ផ្តោតលើប្រអប់ដែលពិបាកទស្សន៍ទាយ (hard examples)
Total Loss
Loss = α·BoxLoss + β·ClsLoss + γ·DFLLoss
α=7.5, β=0.5, γ=1.5 (default weights)

7️⃣ Learning Rate Schedule

Warmup Phase
  • warmup_epochs = 3
  • warmup_momentum = 0.8
  • warmup_bias_lr = 0.1
  • LR កើនឡើងពី 0 → lr0
Main Training
  • lr0 = 0.01 (initial)
  • lrf = 0.01 (final factor)
  • Cosine Annealing Schedule
  • LR decays from 0.01 → 0.0001
📊 LR Formula
lr = lr0 · (1 - cos(π·epoch/epochs)) · (1 - lrf) + lrf

Metrics សម្រាប់វាយតម្លៃ

កំឡុងពេលហ្វឹកហាត់ អ្នកនឹងឃើញតម្លៃទាំងនេះ៖

🎯

Precision (P)

រូបមន្ត
P = TP / (TP + FP)

TP = True Positive (ទស្សន៍ទាយត្រឹមត្រូវ)
FP = False Positive (ទស្សន៍ទាយខុស)

→ ចំនួនទស្សន៍ទាយត្រឹមត្រូវក្នុងចំណោមទស្សន៍ទាយទាំងអស់

🔍

Recall (R)

រូបមន្ត
R = TP / (TP + FN)

TP = True Positive
FN = False Negative (ខកខានមិនឃើញ)

→ ចំនួនទស្សន៍ទាយឃើញវត្ថុពិតទាំងអស់

📊

ការសាកល្បង និងដាក់ឱ្យប្រើប្រាស់ (Testing & Deployment)

លម្អិត

បន្ទាប់ពីហ្វឹកហាត់រួច យើងត្រូវ សាកល្បង (Test), វាយតម្លៃ (Evaluate) និង ដាក់ឱ្យប្រើប្រាស់ (Deploy) ម៉ូដែលរបស់យើង។

📊

1️⃣ Validation

  • ប្រើ Validation Set (10-15% of data)
  • ធ្វើកំឡុងពេល training (each epoch)
  • គណនា mAP, Precision, Recall
  • ជួយតាមដាន overfitting
  • អាចធ្វើបានច្រើនដង
💡 ប្រើ model.val() ក្នុងពេល training
✅

2️⃣ Testing (Inference)

  • ប្រើ Test Set (unseen data)
  • ធ្វើ តែម្តង ក្រោយ training
  • វាស់ performance ពិតប្រាកដ
  • មិនប្រើសម្រាប់ tuning
  • ផ្តល់លទ្ធផលចុងក្រោយ
💡 ប្រើ model.predict() សម្រាប់ inference

Pro Tips
  1. Start Small

    ចាប់ផ្តើមពីម៉ូដែលតូច (nano/small) ដើម្បីសន្សំពេល និងធនធាន

  2. Transfer Learning

    ប្រើ Pretrained Weights ដើម្បីរៀនលឿន ទោះបីទិន្នន័យតិចក៏ដោយ

  3. Balanced Dataset

    ត្រូវប្រាកដថាចំនួនរូបភាពក្នុង Class នីមួយៗមានតុល្យភាព

  4. Use Google Colab

    បើគ្មាន GPU ប្រើ Google Colab (ឥតគិតថ្លៃ) ជាមួយ GPU T4

  1. Monitor mAP

    តាមដាន mAP ជានិច្ច ដើម្បីដឹងថាម៉ូដែលកំពុងរៀនបានល្អឬអត់

  2. Test Different Thresholds

    សាកល្បង confidence (0.1-0.9) និង iou (0.3-0.7) ដើម្បីរកតម្លៃល្អបំផុត

  3. Export for Production

    Export ទៅ ONNX/TensorRT សម្រាប់ deployment លឿនជាងមុន

  4. Version Control

    រក្សាទុក model versions ផ្សេងៗ ដើម្បីអាចត្រលប់មកវិញបាន
